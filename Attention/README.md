# German-to-English Machine Translation with Bahdanau Attention

This project implements a Sequence-to-Sequence (Seq2Seq) neural machine translation model with **Bahdanau Attention** to translate German sentences into English. Built from scratch in PyTorch, the model is trained on the **Multi30k** dataset, which consists of German image descriptions and their English translations.

---

## 🏗️ Model Architecture

The architecture consists of an Encoder-Decoder framework augmented with an additive attention mechanism:

```
                  +---------------------------------------+
                  |           Attention Weights           |
                  +-------------------+-------------------+
                                      | (Softmax alignment)
                                      v
+--------------+              +---------------+              +--------------+
|   Encoder    |------------->|  Attention    |<-------------|   Decoder    |
| (Bi-directional GRU)        |  (Bahdanau)   |              | (Single GRU) |
+--------------+              +---------------+              +--------------+
       ^                                                             |
       |                                                             v
 [Source (DE)]                                                 [Target (EN)]
```

### 1. Encoder (Bi-directional GRU)
* Embeds German source tokens and applies dropout regularization.
* Processes sentences using a bi-directional GRU.
* Utilizes PyTorch's `pack_padded_sequence` and `pad_packed_sequence` to skip padding tokens dynamically during RNN execution, which accelerates training speed and stabilizes gradients.
* Concatenates the forward and backward hidden states from the GRU and projects them through a linear layer with a $tanh$ activation function to form the initial decoder hidden state.

### 2. Attention (Bahdanau Attention)
* Computes attention coefficients by calculating the "energy" between the current decoder hidden state and all encoder outputs.
* Applies a masking operation by setting attention energy weights corresponding to source padding tokens to a large negative number (`-1e10`) so they are ignored after softmax activation.
* Generates a soft-alignment distribution over the source tokens.

### 3. Decoder
* Autoregressively predicts the probability distribution of the next target word.
* Concatenates the embedded previous target token with the attention-weighted context vector as input to the GRU.
* Projects the GRU output, the context vector, and the target embedding to the vocabulary space to generate predictions.

### 4. Seq2Seq Wrapper
* Connects the Encoder and Decoder.
* Implements **Scheduled Teacher Forcing Decay** to balance ground truth supervision and autoregressive prediction during training.

## ⚡ Key Optimizations & Training Best Practices

The codebase incorporates several production-grade optimization techniques:

* **Efficient Collation**: A custom `collate_fn` handles tokenization (via Spacy), vocabulary index mapping, sequence padding (via `pad_sequence`), and encapsulates sequence bookending with `<sos>` and `<eos>` tokens.
* **L2 Regularization**: Weight decay (`weight_decay=1e-5`) is added to the Adam optimizer to constrain weight magnitudes and mitigate overfitting.
* **Scheduled Teacher Forcing**: The teacher forcing ratio decays linearly over training epochs (e.g., from `0.5` down to `0.1` or `0.3`) to expose the model to its own errors, resolving *Exposure Bias*.
* **Early Stopping & Checkpoint Saving**: Tracks validation loss and automatically saves the best generalization parameters to `best-model.pt`. It automatically reloads these parameters before test-set evaluation.

## 📈 Training Performance

A 60-epoch training run on the Multi30k dataset demonstrates the following convergence timeline:

* **Epoch 1**: `Train Loss: 4.19` (PPL: `66.05`) | `Val. Loss: 3.60` (PPL: `36.71`)
* **Epoch 10 (Optimal Point)**: `Train Loss: 2.01` (PPL: `7.50`) | `Val. Loss: 3.30` (PPL: `27.19`)
* **Epoch 60 (Overfitted)**: `Train Loss: 1.24` (PPL: `3.48`) | `Val. Loss: 3.86` (PPL: `47.88`)

*Note: Since the validation loss increases after epoch 10 while the training loss decreases, the model is prone to overfitting. The implemented **Early Stopping** checkpoint loader successfully restores the optimal Epoch 10 parameters.*

## 🚀 How to Run

### 1. Requirements
Ensure you have PyTorch, Hugging Face Datasets, and Spacy models installed:
```bash
pip install torch torchdata datasets spacy
python -m spacy download en_core_web_sm
python -m spacy download de_core_news_sm
```

### 2. Execution
Open the notebook and run all cells sequentially:
```bash
jupyter notebook Attention/Machine_translation_attention.ipynb
```
*The notebook automatically detects if a CUDA-capable GPU is available and utilizes it for training.*
